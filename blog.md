# 测试开发秋招备战笔记（中厂AI测试/自动化/测开方向）

## 一、秋招核心目标
- 目标岗位：中厂以上 **AI测试 / 自动化测试 / 测试开发**
- 秋招前目标：**2段及以上实习背书**
- 核心竞争力：技术栈 + 项目实战 + 实习经历 + 简历自洽

---

## 二、必学技术栈 & 知识体系
### 1. 核心基础
- Python 编程 & 手撕代码
- SQL 基础查询（MySQL/PostgreSQL）
- Linux 命令行操作
- 计算机网络、操作系统、数据库
- 算法题（简单/中等）、八股文背诵

### 2. 测试核心技术
- 功能测试流程 & 测试用例设计
- 接口测试（Apifox/Postman/Jmeter）
- UI 自动化（Selenium / Playwright）
- 自动化测试框架（Pytest + Allure）
- 抓包与问题定位（F12/Fiddler/Charles）
- 性能测试（压测、指标分析）

### 3. 工程化工具
- Git 版本控制
- Docker 容器
- Jenkins CI/CD 持续集成
- 自动化项目部署 & 报告生成

### 4. AI 方向（加分项）
- LLM 大模型基础
- RAG 项目开发与测试点
- AI Agent 基本原理
- 模型评估指标（精确率、F1、AUC）

---

## 三、测试核心方法论
### 1. 测试通用流程
业务背景 → 产品需求 → 开发实现 → 测试需求 → 测试设计 → 基础测试 → 进阶测试 → 报告输出

### 2. 测试具体步骤
理解业务 → 明确需求 → 写测试点 → 执行用例 → 记录结果 → 输出报告

### 3. 测试思维
- 发散性覆盖场景
- 报错→定位→分析→解决
- 模拟用户真实行为，保障产品质量

### 4. 常用工具
Apifox、Jmeter、Xmind、Swagger、Mock、Fiddler、Charles

---

## 四、自动化测试体系
### 1. 接口自动化
Pytest + Requests + Allure 报告 + 数据驱动

### 2. Web 自动化
Selenium（八大定位、三大等待）
Playwright（现代高效方案）
POM 设计模式 + 关键字驱动

### 3. 企业级自动化流水线
Git + Docker + Jenkins + WebHook
自动构建 → 自动执行 → 自动发报告
