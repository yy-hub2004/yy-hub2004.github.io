# 我的核心项目实战
这里是我在测试开发领域的实战项目，全部来自于企业真实业务场景，具备完整的业务闭环与落地成果。

---

## 项目一：Wooshpay 核心支付接口自动化测试框架
**项目周期**：2026.01 - 2026.02  
**技术栈**：Python + Pytest + Requests + YAML + Allure  
**项目角色**：独立负责人

### 项目背景
Wooshpay作为跨境支付服务商，核心支付链路（收银台、入账、发卡、出款等）业务逻辑复杂，版本迭代频繁（每周2-3次），手工回归测试耗时久、重复工作量大，且支付业务对准确性要求极高，手工测试存在漏测风险，亟需通过自动化测试提升回归效率与测试质量。

### 核心目标
搭建高可用、易维护的支付接口自动化测试框架，实现核心支付流程100%场景覆盖，将版本回归测试效率提升60%以上，同时精准拦截支付逻辑错误，规避资金安全风险。

### 技术实现与我的工作
1.  **分层架构设计**：基于Python+pytest+requests构建「数据层-用例层-逻辑层-工具层」的分层自动化架构，实现测试用例、测试数据与业务逻辑的完全解耦，大幅提升框架的可维护性与可扩展性。
2.  **通用工具封装**：封装通用HTTP请求工具类，统一处理接口鉴权签名、参数加解密、请求日志记录等通用逻辑，减少重复代码，提升用例编写效率。
3.  **数据驱动设计**：采用YAML文件实现数据驱动，将正常支付、异常卡号、金额边界、权限异常等多场景的测试数据外置管理，新增/修改用例无需改动代码，大幅降低用例维护成本。
4.  **精准断言策略**：设计多层级断言机制，不仅校验接口响应状态码、核心字段，还增加了数据库资金流水一致性校验，精准把控支付交易的资金安全，减少漏测风险。
5.  **测试报告与持续集成**：集成Allure测试报告，自动生成可视化测试报告，清晰展示用例执行情况、通过率、失败原因；结合Git实现脚本版本控制，可对接Jenkins实现持续集成自动化执行。

### 项目成果
- 实现收银台、Pay In、Payout等核心支付接口100%场景覆盖，框架上线后支撑每周2-3次的版本回归，将单次回归测试时间从数小时压缩至分钟级，**测试效率提升60%以上**。
- 通过多层级断言与全场景覆盖，在多次版本迭代中成功拦截多起潜在支付逻辑错误，有效保障了支付链路的稳定运行，规避了资金安全风险。
- 沉淀了一套可复用的接口自动化测试框架模板，可快速适配其他业务线的接口自动化需求。

## 项目二：Wooshpay 后台系统 UI 自动化测试套件
**项目周期**：2026.01 - 2026.03  
**技术栈**：Python + Pytest + Playwright + PageObject + Allure  
**项目角色**：独立负责人

### 项目背景
Wooshpay后台CRM客户管理系统、MBO审核系统是公司业务运营的核心系统，涉及多角色权限管理、敏感数据审核、全链路数据流转，业务功能更新频繁，每次版本迭代都需要对核心场景进行全量回归，手工回归耗时久、重复性高，且存在人为操作失误导致的漏测风险，严重影响测试效率与业务上线进度。

### 核心目标
独立设计并开发高稳定、易维护的Web端UI自动化测试套件，替代核心业务场景的手工回归测试，解决手工回归低效、用例误报率高的痛点，将核心场景回归效率提升80%以上，脚本执行稳定性达到95%以上。

### 技术实现与我的工作
1.  **技术选型优化**：选用Playwright替代传统Selenium作为自动化驱动工具，天然支持多浏览器、自动等待、元素追踪等能力，从根本上解决了传统Selenium元素定位超时、浏览器兼容性差、用例误报率高等行业痛点。
2.  **PageObject设计模式**：引入PageObject设计模式，将每个页面的元素定位、通用操作封装为独立的页面类，业务用例直接调用页面类的方法，实现页面元素与业务逻辑的分离，大幅提升脚本的可读性、可维护性与复用性。
3.  **稳定性优化**：用Playwright的智能自动等待机制替代传统的固定sleep等待，结合元素可见性、可点击性判断，有效解决页面加载延迟导致的用例失败问题；结合Fiddler抓包辅助定位前后端交互问题，优化异常处理逻辑，降低用例误报率。
4.  **全场景覆盖**：全面覆盖多角色权限校验、审核流程全链路、数据展示准确性、异常场景提示等核心业务场景，实现自动化脚本批量执行，完美适配高频功能迭代的回归测试需求。
5.  **可视化报告**：集成Allure测试报告，自动生成带截图、日志的可视化测试报告，用例失败时可快速定位问题原因，提升问题排查效率。

### 项目成果
- 成功落地UI自动化测试套件，全面替代核心后台场景的手工回归测试，**减少80%以上的重复手工工作量**，大幅缩短版本回归周期，加快了业务上线速度。
- 解决了页面元素定位不稳定、用例误报率高等行业痛点，**脚本执行稳定性达到95%以上**，在版本发布前快速拦截前端渲染、数据交互等问题，保障了后台系统的稳定运行与业务流程通畅。
- 沉淀了一套基于Playwright的UI自动化测试最佳实践，可快速复用至其他Web系统的自动化测试中。
